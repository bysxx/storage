Section 02

프로젝트 배경 및 현황

회사 상황

하파시스 그룹(데스커 모기업)은 AI 도입에 매우 적극적인 기조를 보이고 있으며, 그중에서도 데스커가 그룹 내에서 가장 빠르게 움직이고 있다. 경영진은 AI로 대체 가능한 영역을 모두 대체하겠다는 방향성을 갖고 있으며, 바이브코딩을 포함한 AI 활용을 적극 권장하고 있다.

핵심 배경

  • 그룹사 차원에서 SNS 에이전시 계약 구조가 변화 중
  • 인하우스 전환 추진 — 사람 충원 대신 AI 자동화로 대체 모색
  • 바이브코딩으로 내부 프로토타입을 만들었으나, 본업과 병행하며 개발하기 어려운 상황
  • 클팀은 데스커 업무를 오래 담당해 온 에이전시로, 업무 이해도가 높아 파트너로 적합

프로젝트의 핵심 목표

단순히 콘텐츠를 만드는 것이 아니라, SNS 운영의 시작부터 끝까지 전체 프로세스를 AI 에이전트로 자동화하는 것이 궁극적 목표이다.

Section 03

요구 기능 범위

대시보드 / 콘텐츠 관리

기능 상세 내용 우선순위
콘텐츠 현황 대시보드 월별 발행 계획 대비 진행 상태 (발행 완료 / 대기 / 컨펌 중 등) 한눈에 파악 최우선
계정별 분리 관리 데스커 오피셜, 디퍼 계정을 분리하여 각각의 콘텐츠 파이프라인 관리 최우선
담당자 배정 및 리마인더 콘텐츠별 담당자 지정, 매일 아침 "오늘 나가야 할 콘텐츠" 알림 최우선
월별 스케줄링 캘린더 형태로 콘텐츠 발행 일정 관리 (기존 구글 플래너 대체) 최우선

콘텐츠 제작 자동화

기능 상세 내용 우선순위
AI 콘텐츠 생성 자연어 입력 → 브랜드 가이드 기반 풀 콘텐츠(카피+이미지) 자동 생성 최우선
브랜드 가이드 학습 데스커/디퍼 브랜드 어투, 톤앤매너, 디자인 가이드 학습 적용 최우선
템플릿 기반 이미지 디자인팀 제공 템플릿에 제품 이미지+카피+로고를 자동 배치 중요
AI 배경 이미지 생성 제품 실사 이미지 유지 + 배경(공간/인테리어)을 AI로 생성 중요
아이디어 제안 기능 미라(Mira) 서비스처럼 콘텐츠 아이디어를 자동 제안 희망

운영 자동화

기능 상세 내용 우선순위
자동 메일링 (취합 요청) 전월 20일 자동으로 관련자에게 콘텐츠 소재 취합 요청 메일 발송 중요
컨펌 메일링 최종 담당자에게 콘텐츠 컨펌 요청 자동 발송 중요
메타 성과 데이터 연동 메타 API 통해 성과 데이터 자동 수집 (현재 가능 확인됨) 중요
댓글/DM 자동 응대 초안 컨택센터 기존 답변 DB 기반 자동 응대 초안 생성 검토 필요
이벤트 댓글 취합/당첨자 선정 이벤트 댓글 자동 수집 및 당첨자 추출 희망

자동 업로드에 대한 합의

SNS 채널 직접 업로드 자동화는 계정 차단/밴 위험이 있어 제외. 대신 구글 스프레드시트 등에 콘텐츠를 자동 축적하고, 담당자가 복사-붙여넣기로 수동 게시하는 방식으로 합의. 브랜드 자산 보호가 최우선.

Section 04

현재 개발 현황 (바이브코딩 프로토타입)

데스커 내부에서 상인님이 Claude Code(바이브코딩)를 활용해 약 40% 미만 수준의 프로토타입을 개발한 상태이다. 차장님이 해당 폴더를 인수받아 Claude Code로 구조를 분석하여 파악하였다.

프로토타입 에이전트 구조

프로토타입에서 확인된 구현 사항

  • 에이전트 = 팀장: 전체 워크플로우를 관장하는 오케스트레이터
  • 스킬 = 실무자: 카피 작성, 이미지 합성, 검수 요청 등 개별 기능
  • 룰 = 브랜드 가이드: 브랜드 위키, 제품 레퍼런스, 디자인 규칙 등
  • 피그마 연동, 구글 계정 연동, 이미지 자동 생성까지 구현 시도
  • 브랜드 가이드(어미, 어투, 단어 사용 규칙)가 코딩 파일로 학습됨
  • "프로덕트 폴더는 절대 건들지 마" 등 보호 룰 설정

현재 한계점

  • 디자인 품질이 만족스럽지 못함 — 디자인팀과 인하우스 간 인식 간극
  • 수정 작업에 사람 손이 많이 들어감
  • 메타 API 직접 연동 시 30분마다 토큰 갱신 필요 → 브랜드워치 등 간접 연동 고려
  • 이미지 AI 생성은 로고/텍스트 렌더링 문제로 어려움
  • 본업 병행으로 개발 지속 불가
Section 05

이미지/디자인 관련 논의

합의된 접근 방식

T
템플릿 기반 접근 1차 방향
디자인팀 제작 템플릿 + AI 콘텐츠 합성
디자인팀이 다양한 템플릿을 제작하고, AI가 제품 이미지 + 카피 + 로고를 자동으로 배치. 완전한 AI 이미지 생성이 아닌, 기존 에셋을 활용한 조합 방식. 로고/텍스트 렌더링 오류 위험 없음.
B
AI 배경 생성 추가 가능
제품 유지 + 배경 변환
데스커 제품 실사 이미지를 유지하면서 배경(인테리어, 공간 등)만 AI로 변경. 체험단 이미지 스타일의 "데스크테리어" 이미지 생성. 클팀에서 충분히 가능하다고 확인.

디자인팀과의 간극

AI로 만든 콘텐츠에 대해 디자인팀 일부에서 "데스커답지 않다"는 의견. 데스커 제품이 간결하고 단조로운 특성 때문에 에셋 소스 자체가 부족한 근본적 문제도 있음. 디자인팀이 직접 템플릿을 만들어 제공하는 방향으로 해소 예정.

Section 06

계약 형태 및 비용 구조

데스커 측 구상

기존 SNS 운영 에이전시 계약이 아닌 "AI 프로그램 구축 및 유지보수 계약"으로 재정의. 학습된 폴더(에이전트 시스템)를 하나의 프로그램으로 보고 납품받는 형태.

비용 지불 방식 논의

방식 내용 데스커 선호도
일시불 구축비 초기 구축비를 한 번에 지불하고 프로그램(폴더) 납품 선호 (재경적으로 편함)
구독형 월정액 매달 유지보수비를 정기 지급 검토 가능
분기별 갱신 분기마다 프로그램 갱신 계약 검토 가능
선금 + 잔금 누적 초기 구축비 선금 후, 유지보수비 누적 잔금 정산 실무 선호

결과물 납품 형태

"학습된 폴더"를 하나의 프로그램으로 간주하여 납품. 소스코드/에이전트 시스템 전체를 폴더 형태로 전달. 일부 에이전시에서는 이러한 결과물에 대해 제한(판권)을 두는 사례도 있다는 언급이 있었으며, 클팀 내부에서 별도 검토 예정.

Section 08

핵심 합의 사항 및 액션 아이템

핵심 합의 사항

  • 양측 모두 협업 의지 확인 — 클팀 부대표 "무조건 같이 작업하는 건 의미 있다", 데스커 "레퍼런스로 쓰셔도 된다"
  • 계약 형태: SNS 운영 계약이 아닌, AI 프로그램 구축/유지보수 계약
  • 자동 업로드 제외: 계정 차단 위험으로 스프레드시트 축적 → 수동 게시
  • 이미지 방향: 완전 AI 생성이 아닌 템플릿 기반 + 배경 생성 병행
  • 클팀의 강점: 데스커 업무 이해도 + 개발 역량(뷰저블 개발팀 보유)
  • 데스커가 레퍼런스 사례 제공에 동의
